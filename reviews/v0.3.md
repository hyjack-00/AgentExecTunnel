# Review v0.3

## Scope

Three coupled changes:

1. **Base64-wrapped ssh transport** for `submit_gitbash_ssh.py`: the submitter base64-encodes the user's payload and renders a `ssh HOST "bash -c \"$(echo '<b64>' | base64 -d)\""` trampoline. Every intermediate shell parse layer sees the base64 blob as an atomic literal, so the payload bytes are not parsed by any shell except the final one that runs them. Quoting never degrades no matter how many nested `'`/`"`/`$`/`` ` ``/`\n` the user writes.
2. **Unified envelope transport**: remove `submit_mode` and `target_host` from the task envelope, from `TaskRecord`, from `publish_task` / `submit_task` / `submit_and_wait`. Delete `Executor._execution_command`. Every submitter CLI now renders its own command client-side and submits a single plain string. Executor is mode-agnostic and runs `task["command"]` via `/bin/sh -c`.
3. **Preview stays human-readable**: `write_gitbash_ssh_preview` continues to print `ssh HOST '<payload>'`, not the base64 form. Operators reading the terminal still see the intent of the command.

Availability suite takes a minor adaptation (probes.py's `submit_mode` is now a CLI-dispatch hint, not an envelope field). `command_digest` collapses to a single-argument function. SKILL.md, DESIGN.md, README.md updated.

## Findings

### base64 transport (`_submit_common.render_gitbash_ssh_command`)

- `payload.encode("utf-8")` → `base64.b64encode(...)` → single-line ASCII (Python's `b64encode` does not line-wrap). The encoded string contains only `[A-Za-z0-9+/=]` — shell-safe anywhere.
- `relay_script = f"ssh {shlex.quote(host)} \"bash -c \\\"\\$(echo '{b64}' | base64 -d)\\\"\""`:
  - Outer dquote is processed by the executor's sh once to extract the ssh argv[2]; inside, `\"` and `\$` are bash dquote escapes producing literal `"` and `$`.
  - `\$(…)` keeps the command-substitution inert on the executor side (the executor only forwards it); the `$()` is evaluated by the remote bash after ssh delivers the argv[2] string verbatim.
  - Payload lives inside `'{b64}'` which is literal in every shell context, so the base64 blob is never chewed.
- Parse-count for the payload: 1 (the remote `bash -c <decoded>` is the only layer that reads the user's bytes).
- Preview function `write_gitbash_ssh_preview` is intentionally NOT updated to show the base64 form. It prints the old human-readable `ssh HOST 'payload'` shape. A comment in SKILL.md plus a banner in DESIGN.md explicitly calls out "preview ≠ wire".

### Unified envelope (`protocol.py`, `submitter.py`, `_submit_common.py`, `executor.py`)

- `TaskRecord` drops `submit_mode` and `target_host`. `to_envelope()` accordingly. Version bumped to `v0.3` in `PACKAGE_VERSION`.
- `command_digest(command)` — single arg. Old callers that passed (command, submit_mode, target_host) all updated. Digest computation still `sha256(json({"command": ...}, sort_keys=True))`.
- `publish_task()` and `submit_task()` no longer accept `submit_mode` / `target_host`. Both accept optional `metadata: dict`.
- `submit_and_wait()` signature: `(label, command, timeout_seconds, metadata=None)`. Was `(label, command, submit_mode, timeout_seconds, target_host=None)`.
- `Executor._execution_command` method deleted. Worker directly does `subprocess.Popen(task["command"], shell=True, ...)`.
- Unused `import shlex` in `executor.py` removed.
- `submit_gitbash_ssh.py`: now calls `render_gitbash_ssh_command` and submits the rendered `relay_script` with `metadata={"ssh_host": args.host}`.
- `submit_powershell_ssh.py`: now calls `render_ssh_command` and submits the rendered powershell cmdline.
- `submit_gitbash.py` / `submit_powershell.py`: just submit their raw payload.
- Constants `MODE_RELAY` / `MODE_SSH` removed from `_submit_common.py`.

## Residual risk

- **Payload size**: base64 expands by 4/3. The entire `relay_script` becomes the executor's sh -c argv — `ARG_MAX` (typically 128KB–2MB on Linux) puts an upper bound. For tasks that encode a multi-MB script, `submit_files.py` + reference is the right path.
- **NUL bytes in payload source**: base64 can carry NUL, but bash refuses scripts with NUL in source bytes. Any transport over `bash -c` has the same limit; unavoidable.
- **Remote host needs `base64`**: universally available on Linux/macOS/BSD/busybox. Missing only on ultra-minimal images.
- **`$0` / `$BASH_SOURCE` in the decoded script**: set to `"bash"`, not a filename. Most scripts don't care; advanced ones that read `$BASH_SOURCE` would need a different plumbing.
- **Error line numbers** in decoded scripts lose file mapping. Debug by reproducing locally via `echo '<b64>' | base64 -d | bash -n`.

## Deferred to post-v0.3 (in PLAN.md)

- Whether to also base64-wrap non-ssh `submit_gitbash.py` — currently its outer single-quote + 1 executor shell parse is enough for typical payloads.
