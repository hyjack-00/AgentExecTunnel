# Review v0.0.5

## Scope

- switch default runtime roots to the checked-out repository-local submodules
- switch `.gitmodules` to explicit GitHub HTTPS URLs
- repair bootstrap so stale local file remotes can be localized into `var/local_remotes/`
- fix data submodule ignore rules so runtime task / ack / result files are actually trackable
- document the git-operation model with explicit submitter / executor sequence diagrams

## Findings

- No blocking findings remain for `v0.0.5`.
- The previous repository default paths still implied `/workspace` sibling repos; that was inconsistent with the intended repo-local submodule layout and is now corrected.
- The checked-in submodule definitions are clearer and less ambiguous with explicit HTTPS URLs than with relative `../...` URLs.
- The main source of confusion around "same machine should be safe" was not file overlap, but shared-worktree git operations:
  - both roles run `git_sync()`
  - `git_sync()` performs `fetch + checkout -B + reset --hard`
  - this means the unsafe boundary is a shared local worktree, not the remote repo
- The supported concurrency model is now explicitly documented as:
  - separate submitter/executor clones
  - same forward/backward remotes
- The submodule data repos had ignore rules that hid runtime protocol files from git; removing those rules was required for real submit/execute visibility.

## Residual Risk

- `submitter.wait_for_result()` still refreshes the local backward clone by resetting it to `origin/main`; this is correct for separate-clone operation but remains unsuitable for a shared worktree model.
- ACK-only tasks still require manual repair; there is no automatic takeover.

## Decision

Release `v0.0.5`.
