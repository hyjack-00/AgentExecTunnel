# Review v0.0.4

## Scope

- add repository-local skill for the AgentExecTunnel architecture
- rewrite design docs around sequence diagrams and synchronization rules
- remove the outdated single-submit assumption
- add multi-submitter publish coverage
- add fresh-clone startup coverage for a new machine layout
- add local fake-relay roundtrip coverage
- add and run the 30-second local burst stress path
- rename submodule working directories to `agent_forward/` and `agent_backward/`

## Findings

- No blocking findings remain for `v0.0.4`.
- The old monolith-era forward window limit was the main blocker for multi-submitter burst behavior; removing it and relying on fetch/rebase/push retry fixed the publish bottleneck.
- The design now explicitly documents:
  - backward as the only authority for completion
  - relay and ssh as one runtime class of work
  - weak-network behavior
  - separate working clones as the recommended deployment rule
- Fresh-clone bootstrap and `executor/run_executor.py --once` now pass in a temp layout that simulates a new machine checkout.
- The local fake-relay roundtrip and 30-second burst paths both passed against real local bare remotes.

## Residual Risk

- ACK-only tasks still require manual repair; there is no automatic takeover.
- Availability still needs richer SSH preset coverage.
- CLI-level tests for every submit/repair wrapper are not complete yet.

## Decision

Release `v0.0.4`.
