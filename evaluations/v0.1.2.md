# Evaluation v0.1.2

## Changes

- `agent_exec_tunnel/executor.py`: scan log emits `claimed=/running=` with zero fields suppressed; idle scans stay silent
- `executor/run_executor.py`: new `preflight_check()` bails out with a bootstrap hint when `agent_forward/.git` or `agent_backward/.git` is missing
- `tools/bootstrap_repos.py`: wraps `CalledProcessError` into a `SystemExit` with the failing command and its stderr
- `tools/run_burst_live.py` and `tools/run_burst_local_relay.py`: `load_remote_urls()` returns the `branch` alongside URLs and threads it through `attach_repo_clones` / `sync_repo_to_remote_url`
- `tests/test_cli_entrypoints.py`: added `test_bootstrap_ensure_repo_rejects_non_empty_non_git_dir`
- `DESIGN.md`: `Task State Model` replaced with explicit state table, `stateDiagram-v2`, and `Main-Loop Reconciliation` rules; `Interrupted Finalize` aligned with the new model; `Why Shared Worktree Is Unsafe` sequence diagram removed (prose retained)
- `README.md`: `Migrating from the old submodule layout` section added
- `agent_exec_tunnel/config.py`: `PACKAGE_VERSION` bumped to `v0.1.2`
- `PROGRESS.md`: closed items E–I plus scan-log polish; logged open items for main-loop reconciliation implementation + tests

## Coverage

- unit: `tests/test_cli_entrypoints.py::CliEntrypointTests::test_bootstrap_ensure_repo_rejects_non_empty_non_git_dir`
- unit: existing `test_executor_flow.py` cases exercise the new scan-log output (observed `scan claimed=1`, `scan running=1`, `scan claimed=1 running=1 finished=1`-style lines in captured output, with zero-suppression working)
- full `python3 -m unittest discover -s tests`: 41 passing + 1 skipped + 1 pre-existing order-sensitive error in `test_multiple_submitters_can_publish_without_forward_window_limit` (passes in isolation; unrelated to this release)

## Not Run In This Change

- Live burst (`tools/run_burst_live.py`) was not re-executed; its only behavior change is the branch propagation, which is deterministic from the URL-loading path.
- Windows host executor startup on a machine with stale `.git/modules/agent_forward` was not verified; the README cleanup recipe is provided but not empirically confirmed on that host.
- Orphan-`running` reconciliation is design-only in this release; no runtime test exercises "restart an executor that previously ACK'd but never finalized, observe it emit `stale` after `timeout_seconds`". Added to `PROGRESS.md` as a blocking item for the next release.

## Verification Targets For Next Release

- re-run `tools/run_burst_local_relay.py` end-to-end and confirm the new scan log reads cleanly under load (idle silence, busy counts)
- re-run the `unittest discover` suite and verify `test_multiple_submitters_can_publish_without_forward_window_limit` ordering is fixed or isolated
