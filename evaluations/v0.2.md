# Evaluation v0.2

## Changes

- **New transport**: `agent_exec_tunnel/ntfy_transport.py` (stdlib `urllib`) holds `publish` (bounded), `publish_forever` (infinite, honors `Retry-After`, fresh-connection per attempt), `poll_since`, `poll_loop`, `wait_for`, `seed_seen_ids`.
- `agent_exec_tunnel/submitter.py`: `publish_task` + `wait_for_result` route through ntfy; outer retry collapsed to rely on `publish()`'s inner retry; `wait_for_result` adds `submit_timeout_grace_seconds` on top of the caller deadline.
- `agent_exec_tunnel/executor.py`: main thread runs `poll_loop`; each claimed task gets its own `threading.Thread`; worker calls `_finalize_result → _publish_result → publish_forever`. Class shrinks from 588 LOC to ~260 LOC. `GitWriter` / `_reconcile_orphan_stale` / `_recover_from_backward` / `startup_scan*` / `scan_recent*` / `_git_sync_once` / `_bucket_glob` all deleted.
- `agent_exec_tunnel/protocol.py`: `AckRecord` + `ack_path` + `result_path` + `task_path` + `hour_bucket_parts` + `iter_hour_buckets` deleted. `TaskRecord` / `ResultRecord` lose `forward_task_path`; `to_envelope()` replaces `to_json()` with a `kind` discriminator. `new_task_id()` uses 64-bit token_hex jitter.
- `agent_exec_tunnel/config.py`: `default_timeout_seconds` 512 → 300; added `ntfy_server_url` / `ntfy_forward_topic` / `ntfy_backward_topic` / `ntfy_poll_since` / `ntfy_poll_base_seconds` / `ntfy_poll_jitter_growth` / `ntfy_poll_jitter_floor` / `submit_timeout_grace_seconds`; removed `backward_root` / `executor_backward_write_root` / `executor_poll_*` / `submit_poll_interval_seconds` / `steady_scan_hours` / `startup_scan_hours`. `PACKAGE_VERSION` → `v0.2`.
- `submitter/_submit_common.py`: `_poll_for_result` is a thin wrap of `ntfy_transport.wait_for`; `timeout_exit` splits "ntfy unreachable" vs "ntfy reachable, executor silent" branches; `CALLER_POLL_*` and `_read_result_payload` deleted.
- `submitter/submit_{gitbash,gitbash_ssh,powershell,powershell_ssh}.py`: `--timeout-seconds` default 512 → 300.
- `executor/run_executor.py`: `preflight_check` is now a warning, not `SystemExit`. Drops `--once` / `--poll-interval-seconds` flags.
- `tools/bootstrap_repos.py` + `agent_exec_tunnel/remotes.py`: backward-repo plumbing deleted. `.gitignore` drops `agent_backward/`.
- Deleted: `agent_exec_tunnel/repair.py`, `tools/repair_task.py`, `tools/run_burst_live.py`, `tools/run_burst_local_relay.py`, `tests/test_burst_local.py`, `tests/test_fake_relay_roundtrip.py`, `tests/test_fresh_clone.py`, `tests/test_integration_repos.py`, `tests/test_submitter_flow.py`, `tests/runtime_helpers.py`.
- `tests/availability/`: `report.py` restored from legacy (`/workspace/agent_forward` v4.5) — hop cards, p50/p95/p99 panel, stage timings, SVG heartbeat, per-probe table with outcome histogram, recent failures, snapshot + HTTP serve. `probe.py` / `probes.py` / `storage.py` / `ssh` shim restored accordingly.
- `DESIGN.md`, `README.md`, `skills/agent-exec-tunnel-submit/SKILL.md`: rewritten for the ntfy message plane + git file plane split.
- `PROGRESS.md` → `PLAN.md`: repurposed from history log to forward-looking todo; history stays in git log.

## Live verification

- Real ntfy.sh round-trip: `AET_GIT_BASH_EXECUTABLE=/bin/bash python3 submitter/submit_gitbash.py --timeout-seconds 30 "echo hello-ntfy"` → `SUBMITTED command_id=…` in <1s, stdout replay within ~7s. Zero git push on the message plane (verified via `git status` on the worktree's forward/backward roots).
- Timeout: `--timeout-seconds 5 "sleep 20 && echo never"` → executor log emits `finalize … status=stale exit=-1` exactly 5s after claim; submitter exits 124 with "ntfy reachable; executor may be down or overloaded, check executor status".
- Restart idempotency: after two completed tasks, `python3 -c "from agent_exec_tunnel.executor import Executor; from agent_exec_tunnel.ntfy_transport import seed_seen_ids; e = Executor(); print(seed_seen_ids(e.ntfy, e.ntfy.backward_topic))"` returns both `task_id`s so the next `run_loop` won't re-dispatch them.
- Availability report: demo run with 20 seeded probe records generated an 8944-byte `report-latest.html` containing all seven legacy sections (hop cards / latency / stage timings / heartbeat SVG / per-probe / recent failures / snapshot file).

## Not run in this change

- 10-minute idle jitter observation (manual, wall-clock): deferred to runtime. The algorithm is deterministic — `jitter_max = min(149s, prev*1.10 + 0.05)` — but the cadence shape under real internet jitter is easier to observe in staging than asserted in tests.
- File-transfer regression against a real GitHub remote: requires a private remote in the test environment; `tools/bootstrap_repos.py` + `submit_files.py` logic is unchanged from v0.1.3 so the risk is bounded.
- Multi-submitter contention against one ntfy topic (N submitters racing publish + wait): the dedup key is `task_id` and `publish` is independent per-submitter, so behavior is well-defined; no integration test spun up yet. Added to the "Verification targets for next release" list below.
- Windows executor path: `_execution_command` for `submit_mode=="ssh"` uses `shlex.quote` which is posix-only on `cmd.exe`; noted in review as a LOW-priority follow-up.

## Verification targets for next release (v0.2.1)

- Multi-submitter integration test: N=10 concurrent submitters against a fake local ntfy server, assert all 10 results return correctly.
- HMAC envelope signing (or private ntfy endpoint) to close the trust gap on world-readable topics.
- Windows-specific quoting path for ssh-wrapped commands.
- Out-of-band stdout/stderr streaming for large-log tasks.
