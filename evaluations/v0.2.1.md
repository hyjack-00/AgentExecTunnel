# Evaluation v0.2.1

## Changes

- `agent_exec_tunnel/protocol.py`: new `AckRecord` dataclass with `to_envelope() → kind:"ack"`.
- `agent_exec_tunnel/executor.py`:
  - `seen_ids` type `set[str] → dict[str, float]`; `_mark_seen` / `_maybe_prune_seen_ids` helpers; `seen_ids_ttl_seconds` lazy prune from hot path.
  - `_is_expired(task)` boundary guard; `_handle_task_envelope` skips expired envelopes and marks them seen.
  - `_publish_ack` new; `_run_task_worker` publishes ACK before `subprocess.Popen`.
  - startup `seed_seen_ids` now also picks up ACK envelopes.
- `agent_exec_tunnel/ntfy_transport.py`: soft import + `truststore.inject_into_ssl()`; `wait_for` gains `match_kind` filter so submitter-side waiters can skip the intermediate ACK envelope.
- `agent_exec_tunnel/submitter.py` + `submitter/_submit_common.py`: both pass `match_kind="result"` to `wait_for`.
- `agent_exec_tunnel/config.py`:
  - `ntfy_poll_since` `"2h" → "30m"`
  - `seen_ids_ttl_seconds: float = 3600.0` (new)
  - `PACKAGE_VERSION` `"v0.2" → "v0.2.1"`
- `DESIGN.md`: new "ACK envelope" section, updated dedup model, boundary-guard subsection, updated trade-offs (ACK narrows but does not close the re-run window).
- `VERSION`: `v0.2 → v0.2.1`
- tests: `test_executor_flow.py` reshaped — envelope fixture uses live `created_at` (else it would always hit the expiry guard); new tests for ACK ordering, expired-envelope skip, TTL prune, ACK shape.

## Live verification

- Real ntfy.sh E2E: heavy earlier testing tripped the anonymous rate limit (HTTP 429 on both poll and publish). Observed the code behave correctly under the limit:
  - Worker's `publish_forever` ACK/result loop honored the 429 `Retry-After` header and eventually succeeded ("ntfy publish_forever recovered topic=agent-backward-285 attempts=13" in the log).
  - Executor finalized the task (`finalize … status=done exit=0 published=True`) after ntfy recovered.
  - Main poll thread grew jitter on error and did not self-DoS.
  - Submitter timed out per its own deadline (300s + 15s grace); this is expected when the backward publish path is being actively rate-limited well beyond the submitter's wait budget. Not a correctness issue — the executor finished and the result envelope is durable on the topic.

## Not run in this change

- Wall-clock soak test of `seen_ids` pruning over 24h. The TTL prune path is unit-tested; runtime observation is a nice-to-have.
- Private-ntfy / authenticated-topic deployment. Still deferred to the trusted-environment MVP posture.

## Verification targets for next release

- Under a private ntfy instance (no 429 pressure), rerun multi-submitter contention to confirm ACK ordering holds under true burst.
- Consider adding `status:"progress"` heartbeat envelopes for long-running tasks, and update `match_kind` filter semantics accordingly.
